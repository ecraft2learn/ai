Also, by clicking on the orange i, the user can change the name of the variable as seen in the calling script (although the name hasn’t changed inside the block’s definition). This kind of variable is called an upvar for short, because it is passed upward from the custom block to the script that uses it. Note about the example: for is a primitive block, but it doesn’t need to be. You’re about to see (next section) how it can be written in Snap ! . Just give it a different name to avoid confusion, such as my for as above.
