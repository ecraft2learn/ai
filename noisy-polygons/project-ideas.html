Project ideas for Noisy Polygons

  1. After training you can use 'R' to rate the best or worst of 10 new noisy polygons. Change the program to use a bigger number. With a big number are the noisy polygons much better (or worse)? (Warning: large numbers may make it pretty slow.) Does the model predict better if trained with random noisy polygons, best ones, worst ones, or a mixture?
  2. Does it get better with more ratings? See if it then predicts better.
  3. Change draw noisy polygon to draw something else. Add more parameters used in drawing. Or remove some.
  4. Get different people to add their own ratings. How well does it do?
  5. Type 's' followed by 'i' to search for better models. If you decrease the loss do you see an improvement in predictions?
